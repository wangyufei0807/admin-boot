Problem

AuthServiceImpl has 6 direct dependencies including 2 mappers bypassed through the service layer. Cross-cutting concerns (JWT, password encoding, permissions) are injected into business logic.

Solution

Extract token management to TokenService, move mapper access behind service seams, inject only authenticated-user representation (LoginUser) rather than raw infrastructure.

Wins

  • ✓ Locality: login bugs stay in one module
  • ✓ Leverage: test auth without Redis/JWT stack
  • ✓ Interface shrinks: 3 methods vs 6 deps
  • ✓ Mappers behind service seam

Remove Parallel Context — Unify User Resolution

Strong mock

UserContext · SecurityContextHolder · UserContextFilter · PermissionAspect · LogAspect

Before

SecurityContextHolder
UserContext (ThreadLocal)
Filter
Aspect
Controller

After

Single Source
SecurityContext
Authentication.getPrincipal()

Problem

UserContext duplicates Spring Security's SecurityContextHolder. Both hold LoginUser, creating two ways to resolve the current user with no clear winner.

Solution

Delete UserContext and UserContextFilter. Replace all UserContext.getUser() calls with SecurityContextHolder.getContext().getAuthentication().getPrincipal().

Wins

  • ✓ Single source of truth for current user
  • ✓ Delete 3 files, modify 4
  • ✓ Testability: mock SecurityContext
  • ✓ Removes ThreadLocal state

Problem

BaseController is never extended. BaseService adds 3 passthrough methods. Constants duplicates enum values. Utility classes wrap JDK with no value-add.

Solution

Wins

  • ✓ No callers affected (BaseController)
  • ✓ Complexity concentrated, not hidden
  • ✓ Cleaner navigation

Consolidate Permission Checking — Use Spring Security

Worth exploring ports & adapters

Before

  classDef leak stroke:#dc2626,stroke-width:2px;

After

Framework
@PreAuthorize
MethodSecurityInterceptor

Problem

PermissionAspect re-implements Spring Security's @PreAuthorize logic. Two parallel authorization mechanisms can drift or conflict.

Solution

Enable @EnableMethodSecurity, migrate @RequiresPermissions to @PreAuthorize("@ss.hasPermi()"), extract hasPermi to a component method.

Wins

  • ✓ Leverage: framework owns security
  • ✓ Delete PermissionAspect
  • ✓ Consistent with Spring ecosystem
  • ✓ Less custom code

Problem

LogAspect handles serialization, IP extraction, user context, and DB insert in one 170-line class. IP extraction duplicates IpUtils.

Solution

Extract HttpRequestResolver for request extraction, IpResolver for IP resolution (delegates to IpUtils), create OperationLogService for DB operations.

Wins

  • ✓ Locality: each concern isolated
  • ✓ Test IP parsing without aspect
  • ✓ Async log writes possible
  • ✓ Removes IpUtils duplication
